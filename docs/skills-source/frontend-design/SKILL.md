# Frontend Design

## Overview
Creative-driven frontend design guidance for interfaces that need personality and visual distinction. Use it to define a clear visual direction early, then turn that direction into buildable choices for typography, color, composition, and motion.

## When to Use
- Marketing pages, landing pages, branded flows, or polished product surfaces that need a stronger point of view
- Requests for unique visuals, stronger personality, richer motion, or a more memorable interface
- Cases where a plain system-compliant layout meets requirements but does not create the intended impression

## Core Pattern
- Decide the visual thesis before choosing individual styles; name the mood or brand impression in a short phrase
- Pick typography, color, motion, and spatial composition as one coherent system so each choice reinforces the others
- Translate the thesis into buildable rules: type scale, spacing rhythm, palette roles, accent treatments, and motion cues
- Avoid generic AI-looking defaults such as interchangeable card grids, random gradients, or trend-driven decoration without purpose
- Push visual identity without breaking the product: keep hierarchy, readability, responsive behavior, and implementation scope intact

## Quick Reference
- Priorities: visual thesis -> typography -> color -> composition -> motion -> responsive polish
- Inputs: brand intent, audience, content density, constraints, and existing product language
- Outputs: a named direction, palette roles, type pairing, composition rules, and motion cues that engineering can implement directly

## Common Mistakes
- Chasing novelty without a coherent concept or clear brand reason
- Using flashy effects, dense overlays, or decorative motion that weakens readability
- Defaulting to generic gradient-and-card aesthetics instead of making a deliberate visual choice
- Creating a direction that looks good in a static mock but is fragile to build, scale, or maintain
- Ignoring mobile layout, content growth, or reduced-motion needs
