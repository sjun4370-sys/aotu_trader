# UI UX Pro Max

## Overview
System-driven frontend design guidance for structured products such as dashboards, admin tools, and component libraries. Use this when the work needs clear hierarchy, reusable patterns, accessibility coverage, and token-driven consistency across screens.

## When to Use
- Enterprise products, dashboards, admin panels, dense forms, tables
- Design systems, component libraries, token systems
- UX reviews, accessibility reviews, information hierarchy fixes
- Cases where consistency, reuse, and predictable behavior are mandatory

## Core Pattern
- Start from information structure and task flow
- Define semantic tokens before component detail
- Prefer reusable patterns over one-off visual flourishes
- Validate accessibility, touch targets, states, and responsiveness

## Quick Reference
- Priorities: hierarchy first; accessibility always; consistency before polish; responsiveness verified before handoff
- Inputs: product type, primary tasks, constraints, existing patterns, accessibility requirements
- Outputs: semantic tokens, layout rules, component states, interaction rules, UX guardrails

## Common Mistakes
- Starting from decoration before hierarchy
- Mixing semantic and decorative tokens
- Inconsistent spacing, states, or component behavior
- Skipping keyboard and contrast checks
